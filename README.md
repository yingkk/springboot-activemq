# springboot-activemq
Queue:
     1.有多个QueueConsumer时，
                  默认FIFO,消费者进行轮循；
                  默认preFetch limit为1000，可针对不同的消费者设置；
     注：如果一个消费者被允许可以聚集大量的未被确认的消息，会导致其他消费者无事可做； 
        同时，如果这个消费者出错的话，会导致大量的消息不能被处理，直到消费者恢复之前。
     即：当有一个消费者的时候，预取限制可以设置为一个相当大的值，
         当有一组消费者的时候，预取限制可以设置为小一点的数字，0或1（注：设置为0表示consumer每次主动向activeMq要求传输最大的数据量，而不是被动的接收消息）；
